HTTP协议

web/网页

操作系统，中间件，编程语言，数据库

静态网页（几乎没有渗透测试的可能性）

HTML，js

无法调用系统资源，前端用户可以直接看到源代码

常用于博客、政府宣传网页。

动态网页

php，java，与数据库进行交互

中间件服务器（对外进行服务时需要）

Apache：适用于轻量级的web服务，php。

不适合访问量过大的场景，多线程场景

nginx：可实现负载均衡，反向代理

tomcat：专门为了javaweb设计的

数据库

MySQL，Redis

http协议

优点：快

请求报文分析

http请求：由请求行，请求头，请求正文三部分组成

请求行：HTTP 报文的第1行，由空格字符分成三部分

请求方法：GET

资源路径：/admin

协议/版本：HTTP/1.1

请求头：从请求报文第2行开始到第1个空行为止之间的内容

请求正文：第1个空行开始以后的所有内容

POST 传参

⚠️GET方法没有请求正文；POST有

响应报文分析

响应报文由状态行、响应报头、响应正文三部分组成

状态行：响应报文的第1行

协议/版本：HTTP/1.1

响应状态码：200

描述短语：OK

响应报头：响应报文第二行开始到第1个空行为止的所有内容，

其中包含了关于 HTTP 响应的重要字段。

响应正文：响应报文从第 个空行开始到最后的所有内容。 服

务器返回资源的内容，即浏览器接收到的 HTML 代码
